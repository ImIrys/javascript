Camel Case

No JavaScript usa-se muito o termo Camel Case, principalmente quando estamos declarando funções e variáveis.

No Camel Case, ao criar uma variável que possui mais de uma palavra faz-se o seguinte:

  1. A palavra que representa essa variável será escrita tudo junto, sem espaço, sem traços.
  2. Com exceção da 1ª palavra, cada palavra seguinte será escrita com a 1ª letra em maiúscula.
  3. Exemplo: criandoVariavelExemplo

Upper Snake Case

Basicamente, todas as letras em maiúscula e as palavras são separadas por underline "_".

De acordo com as boas práticas, o Upper Snake Case é utilizado para constantes.

Variáveis

Para declarar valores dinâmicos, que podem ser modificiados ao longo do ciclo de vida do código, podemos utilizar as seguintes palavras reservadas do JavaScript: Var e Let

Ao codar, o ideal é utilizar o let.

Let

Ela tem o Escopo de Bloco

Exemplo:
var a = 1;
var b = 2;

if (a === 1) {
var a = 11; // o escopo é global
let b = 22; // o escopo é dentro do bloco if

console.log(a); // 11
console.log(b); // 22
}
console.log(a); // 11
console.log(b); // 2

Var

Ela tem o Escopo Global

Exemplo:
var a = 1;
var b = 2;

if (a === 1) {
var a = 11; // o escopo é global
let b = 22; // o escopo é dentro do bloco if

console.log(a); // 11
console.log(b); // 22
}
console.log(a); // 1
console.log(b); // 2

Hoisting

Atribuição de um valor a uma variável antes de declará-la. Isso só acontece com a palavra var (quando falamos de atribuição de valores, pois as funções também possuem essa propriedade e funcionalidade).

Exemplo usando var
numberOne = 1; // é uma atribuição

console.log(numberOne + 2);

var numberOne; // declaração da variável veio depois. No Hoisting, quando se tem uma variável global, o JavaScript leva para cima, automaticamente, a declaração da variável que veio depois.

Exemplo usando let
numberOne = 1;

console.log(numberOne + 2);

let numberOne; // não vai rodar, dará erro. Pois, o let não possui a funcionalidade que existe na var. Sempre que tiver o let é necessário declarar antes.

let numberOne;

numberOne = 1;

console.log(numberOne + 2);

Redeclaração e Reatribuição

Só acontece com a var (não podemos redeclarar o let, porém podemos reatribuir)

Qual a diferença entre Declaração e Reatribuição?

Declaração é escrever var ou let mais o nome da variável e igualar a alguma coisa ou igualar a nada.

Reatribuição é só o nome da variável sem a palavra reservada na frente (var ou let) e igualar aum outro valor.

Constantes

  • Declaradas em UPPER_SNAKE_CASE;
  • Possuem Escopo de Bloco;
  • Não faz Hoisting;
  • Não pode ter seu valor redeclarado e reatribuido, pois, por ser uma constante, seu valor não muda.

Exemplo:
const FIRST_NAME = "Fiorela"; // Declaração em UPPER_SNAKE_CASE.

console.log(FIRST_NAME);
